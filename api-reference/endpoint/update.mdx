## Overview

This endpoint updates an existing user's information. Use PUT for full updates or PATCH for partial updates.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user to update
</ParamField>

## Body Parameters

<ParamField body="email" type="string">
  User's new email address (must be unique)
</ParamField>

<ParamField body="name" type="string">
  User's new full name
</ParamField>

<ParamField body="role" type="string">
  User's new role
  
  Options: `user`, `admin`, `moderator`
</ParamField>

<ParamField body="password" type="string">
  New password (minimum 8 characters)
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="object">
  The updated user object
  
  <Expandable title="User Object">
    <ResponseField name="id" type="string">
      Unique identifier for the user
    </ResponseField>
    
    <ResponseField name="email" type="string">
      User's email address
    </ResponseField>
    
    <ResponseField name="name" type="string">
      User's full name
    </ResponseField>
    
    <ResponseField name="role" type="string">
      User's role
    </ResponseField>
    
    <ResponseField name="updated_at" type="timestamp">
      When the user was last updated
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL (PUT)
curl -X PUT https://api.example.com/v1/users/123 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Updated",
    "email": "john.updated@example.com"
  }'
```

```bash cURL (PATCH)
curl -X PATCH https://api.example.com/v1/users/123 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Updated"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.example.com/v1/users/123', {
  method: 'PATCH',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'John Updated'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.patch(
    'https://api.example.com/v1/users/123',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    },
    json={
        'name': 'John Updated'
    }
)

data = response.json()
print(data)
```

</CodeGroup>

## Example Response

```json
{
  "success": true,
  "data": {
    "id": "123",
    "email": "john.updated@example.com",
    "name": "John Updated",
    "role": "user",
    "updated_at": "2024-02-03T21:45:00Z"
  },
  "message": "User updated successfully"
}
```

## Error Responses

<ResponseExample>
```json 404 - Not Found
{
  "success": false,
  "error": {
    "code": "user_not_found",
    "message": "User with ID 123 not found"
  }
}
```

```json 400 - Bad Request
{
  "success": false,
  "error": {
    "code": "validation_error",
    "message": "Invalid email format"
  }
}
```

```json 409 - Conflict
{
  "success": false,
  "error": {
    "code": "email_exists",
    "message": "Email is already in use by another user"
  }
}
```
</ResponseExample>

<Info>
  Use PUT when updating all fields, PATCH when updating specific fields only
</Info>
