## Overview

This endpoint permanently deletes a user from the system.

<Warning>
  This action is permanent and cannot be undone. All user data will be deleted.
</Warning>

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user to delete
</ParamField>

## Query Parameters

<ParamField query="force" type="boolean">
  Force delete even if user has associated data (default: false)
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="message" type="string">
  Confirmation message
</ResponseField>

<ResponseField name="data" type="object">
  Information about the deleted user
  
  <Expandable title="Deleted User Info">
    <ResponseField name="id" type="string">
      ID of the deleted user
    </ResponseField>
    
    <ResponseField name="deleted_at" type="timestamp">
      When the user was deleted
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X DELETE https://api.example.com/v1/users/123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash cURL (Force Delete)
curl -X DELETE "https://api.example.com/v1/users/123?force=true" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('https://api.example.com/v1/users/123', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.delete(
    'https://api.example.com/v1/users/123',
    headers={'Authorization': 'Bearer YOUR_API_KEY'}
)

data = response.json()
print(data)
```

</CodeGroup>

## Error Responses

<ResponseExample>
```json 404 - Not Found
{
  "success": false,
  "error": {
    "code": "user_not_found",
    "message": "User with ID 123 not found"
  }
}
```

```json 409 - Conflict
{
  "success": false,
  "error": {
    "code": "delete_conflict",
    "message": "Cannot delete user with associated data. Use force=true to override."
  }
}
```

```json 401 - Unauthorized
{
  "success": false,
  "error": {
    "code": "unauthorized",
    "message": "Invalid or missing API key"
  }
}
```

```json 403 - Forbidden
{
  "success": false,
  "error": {
    "code": "forbidden",
    "message": "You don't have permission to delete this user"
  }
}
```
</ResponseExample>

## Cascading Deletes

When a user is deleted, the following associated data is also removed:

<AccordionGroup>
  <Accordion title="User Profile" icon="user">
    All profile information and settings
  </Accordion>
  <Accordion title="Sessions" icon="key">
    All active and expired sessions
  </Accordion>
  <Accordion title="API Keys" icon="code">
    All API keys associated with the user
  </Accordion>
  <Accordion title="Activity Logs" icon="clock">
    All activity and audit logs (optional, based on retention policy)
  </Accordion>
</AccordionGroup>

## Best Practices

<Tip>
  Consider implementing a soft delete mechanism for user accounts to allow recovery
</Tip>

<Note>
  Always verify the user ID before deletion to prevent accidental data loss
</Note>
